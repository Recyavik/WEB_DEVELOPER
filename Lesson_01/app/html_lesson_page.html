Урок 1 по HTML

HTML как фундамент любой страницы

Эта страница насыщена примерами базовых HTML-элементов, из которых строится почти любой сайт: шапка, меню, основной контент, боковая колонка, списки, карточки, таблицы, формы, кнопки, ссылки и подвал.

Важная мысль урока: HTML задаёт структуру и смысл, CSS оформляет внешний вид, а JavaScript отвечает за поведение. Поэтому сначала мы учимся правильно раскладывать блоки по местам, а уже потом делаем страницу красивой и интерактивной.

1. Минимальная структура HTML-документа

Ниже показан каркас, без которого не обходится практически ни одна современная веб-страница. Даже если документ очень маленький, у него всё равно есть html, head и body.

Минимальный шаблон HTML
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Моя страница</title>
  </head>
  <body>
    <h1>Привет, HTML!</h1>
  </body>
</html>
  • DOCTYPE включает современный стандарт HTML5.
  • html является корневым элементом всего документа.
  • head содержит служебные данные, невидимые в основной области страницы.
  • body содержит весь видимый интерфейс.

2. Как раскладывать виджеты по странице

Когда мы говорим о размещении элементов, сначала нужно мыслить не кнопками и картинками, а крупными смысловыми областями. Такая привычка помогает не сваливать всё в беспорядочный набор div-блоков.

<header>
<nav>
<main>
<aside>
<footer>
  1. Сначала выделяем шапку страницы: логотип, название, главное сообщение.
  2. Затем размещаем навигацию: ссылки на разделы, меню, якоря.
  3. После этого строим main — центральную область с основным материалом.
  4. Если есть второстепенный контент, используем aside.
  5. Внизу страницы ставим footer с контактами, авторством и служебной информацией.

3. Основные элементы, без которых почти не обходится страница

Заголовки

Теги h1-h6 задают иерархию текста: от главного заголовка к более мелким подзаголовкам.

Абзацы

Тег p хранит обычный текст. Он нужен почти на любой информационной странице.

Ссылки

Тег a связывает страницы между собой, ведёт на другие разделы, документы и внешние сайты.

Списки

Списки через ul и ol подходят для меню, преимуществ, инструкций и шагов.

Изображения

Тег img добавляет картинку. Атрибут alt обязателен по смыслу и доступности.

Формы

Через form, input и button пользователь отправляет данные.

5. Таблица как пример структурированных данных

Таблицы нужны не для общего макета страницы, а для действительно табличных данных: сравнений, прайсов, расписаний, отчётов и характеристик.

Элемент Зачем нужен Пример
header Шапка страницы или раздела Логотип, название сайта, контакты
nav Навигация по разделам Меню, хлебные крошки, вкладки
main Главный смысловой контент Статья, каталог, лендинг, карточки
form Сбор и отправка данных пользователя Поиск, заявка, регистрация

6. Формы, поля ввода и запросы

В чистом HTML нет циклов и условий, но есть очень важный механизм общения пользователя со страницей — формы. Через форму можно отправить GET- или POST-запрос.

7. Есть ли в HTML циклы и условия?

Короткий ответ: нет, в чистом HTML их нет. HTML — это язык разметки, а не язык программирования. Он описывает, какие элементы существуют на странице, но сам по себе не умеет говорить: «повтори карточку 5 раз» или «покажи блок только при таком условии».

  • Повторение через циклы обычно появляется в шаблонизаторах или JavaScript.
  • Условия if/else тоже появляются вне чистого HTML.
  • Запросы в HTML чаще всего связаны со ссылками и формами.
  • Медиа-запросы относятся к CSS, а не к HTML.

8. Что ученик должен запомнить после этой страницы

  1. Любая страница начинается с базового каркаса: doctype, html, head и body.
  2. Размещение элементов лучше строить от крупных зон к мелким виджетам.
  3. Семантические теги делают код понятнее людям, браузерам и поисковикам.
  4. Самые частые элементы страницы: заголовки, текст, ссылки, списки, изображения и формы.
  5. HTML не содержит циклов и условий, но отлично задаёт основу для дальнейшей логики.

Подсказки начинающему

  • Сначала думай о смысле блока, а потом о его внешнем виде.
  • Не размещай всю страницу только через бесконечные div без причины.
  • Проверяй вложенность тегов: открытие и закрытие должны совпадать.
  • Всегда добавляй понятный alt для важных изображений.
  • У формы почти всегда должны быть label и name у полей.
